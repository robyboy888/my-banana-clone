## 选题推荐（3–5 个，含优先级）

### 选题 01（P0）：“开源 Agent 工作流周报”怎么变成你的内容护城河？

- 标题方向：
  - 《GitHub Agentic Workflows 周报解读：AI 编程工作流进入“产品化周更”时代》
  - 《把开源周报变成 7 天连载：复刻一套 AI 编程 Agent 流水线》
- 目标受众：程序员创作者、独立开发者、ToB 技术内容运营、AI 编程工具重度用户
- 切题角度：从“新闻转述”升级为“**可复刻**”：挑 1 个版本更新点→复现→给出模板与坑位
- 爆点：开源 + 可复刻 + 有产出（PR/脚手架/模板），极易收藏与转发
- 内容结构（建议）：
  1) 30 秒：这周更新解决了什么问题（用一句话）
  2) 复现准备：环境/依赖/数据（给最小清单）
  3) 实战：用 1 个小任务跑完整链路（录屏）
  4) 迁移：如何套到你的项目（3 种场景）
  5) 风险：可用性边界、成本、团队协作陷阱
- 痛点：很多人“听懂 Agent”，但**做不出工作流**
- 爽点：给一键可用的模板与提示词，直接提升生产力
- 痒点：想知道“现在最像未来”的 AI 编程方式
- 推荐内容形式：横屏长视频/图文长帖（含 repo 链接 + 操作步骤）
- 可引用来源：
  - GitHub Agentic Workflows Weekly Update（2026-05-25）：https://github.github.com/gh-aw/blog/2026-05-25-weekly-update/

### 选题 02（P1）：别再做“功能清单评测”，做“卡顿/冻结/键盘流”的工具体验评测

- 标题方向：
  - 《为什么你的 AI IDE 总卡？Aiola v0.1.40 这类修复才是真生产力》
  - 《任务看板 + Plan 流：我用快捷键把 Agent 工作流提速 30%》
- 目标受众：效率工具用户、AI IDE/桌面客户端用户、想做工具测评的创作者
- 切题角度：用“可复现的坏体验→修复点→对比测试”讲清楚价值
- 爆点：CPU 100%/diff 冻结/二进制文件卡死——强共鸣
- 内容结构（建议）：
  1) 复现：一个大 PNG / 二进制文件让 diff 卡死（演示）
  2) 解释：为什么会卡（渲染/加载顺序/阻塞）
  3) 解决：新版本怎么做（stub + load order）
  4) 结论：给“选型 checklist”（10 条）
- 痛点：工具再强，卡顿/冻结就不可用
- 爽点：给“选型清单”与测试方法，观众可直接用
- 痒点：想找到“真正能当工作台”的 Agent 客户端
- 推荐内容形式：对比评测短视频 + 配套图文清单
- 可引用来源：
  - Aiola Changelog（v0.1.40，2026-05-24）：https://aiola.app/changelog

### 选题 03（P2）：“云端 Agent + Slack/Webhook + 托管产物”是下一代团队内容生产流水线

- 标题方向：
  - 《把 Agent 接进 Slack：从消息触发到产物回传的一套内容生产流水线》
  - 《AI 不止写稿：用 Webhook 订阅事件，把 Agent 变成自动化同事》
- 目标受众：团队内容负责人、增长/运营、ToB SaaS 关注者、自动化爱好者
- 切题角度：不讲“AI 多强”，讲“**怎么接入团队工具链**”
- 爆点：消息触发→自动执行→产物可分享（artifact）→可审计（账号归因）
- 内容结构（建议）：
  1) 场景：每天自动生成选题库/日报
  2) 设计：触发器（Slack/Webhook）+ 权限（service account）+ 产物（hosted artifacts）
  3) Demo：用 1 次触发跑通
  4) 风险：误触发/权限/审计/成本
- 痛点：手动协作成本高、产物流转混乱
- 爽点：一条指令/一条消息就产出完整物料
- 痒点：想把“内容生产”做成系统，不靠人肉
- 推荐内容形式：案例向长视频 + 配套流程图
- 可引用来源：
  - Augment Cosmos 26-05-18 Release Notes：https://www.augmentcode.com/changelog

- 目标受众：小红书/视频号/公众号创作者、知识付费从业者、独立产品人

- 爆点：模板可卖、可复用、能直接带来转化
- 推荐内容形式：图文模板包 + 录屏教程
