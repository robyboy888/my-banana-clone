## 今日判断

今天最值得讲的，不是又有哪个模型分数更高，而是 AI 工具正在抢占真实工作的入口位：

1. Anthropic 把 Claude 放进 Slack 频道，AI 不再只是个人聊天窗口，而是团队可共享的“频道成员”。
2. Google 把 `Computer Use` 放进 Gemini 3.5 Flash 公测，浏览器、移动端、桌面端自动操作开始标准化。
3. Figma、Runway、GitHub 同时推进画布、视频和终端工作面，说明“上下文持续”和“多工具联动”比单一模型更有传播性。
4. ElevenLabs 开始把 AI 直接推到广告投放回路里，创作者能讲的不只是内容生产提效，而是内容商业化提效。

如果今天要做内容，建议把主题抬高到：

- 谁在抢 Slack、Terminal、Canvas、Browser 这些高价值入口；
- 创作者为什么应该少看“单次生成效果”，多看“整段流程是否闭环”；
- 哪些能力已经能影响团队协作、短视频改稿、出海广告和知识付费交付。

## 热点分析

### 1. Gemini `Computer Use` 公测：Agent 开始接管浏览器、桌面和移动端动作

- 时效性：高。Google Gemini API changelog 日期为 2026-06-24。
- 事实要点：
  - Gemini API 在 2026-06-24 发布 `Computer Use` public preview。
  - 官方强调支持 browser、mobile、desktop environments。
  - 官方同时强调 configurable safety policies 和 advanced prompt injection detection。
- 对创作者的意义：
  - 这条比“模型更聪明”更容易转成教程和案例，因为它涉及具体动作执行，而不是纯对话。
  - 对做效率工具、自动化工作流、教育训练营的人，可以讲“从会回答到会操作”的迁移。
- 可讲述角度：
  - 浏览器 Agent 离普通团队还有多远。
  - 为什么真正稀缺的是可控执行，而不是多会写提示词。
- 风险与不确定性：
  - 当前是 public preview，不适合夸大成“已经完全稳定可商用”。
  - 真实复杂网站上的成功率、成本和合规边界，仍需更多第三方验证。

### 2. Figma Config 2026：AI 不是外挂，而是直接长在设计画布上

- 时效性：高。Figma Blog 多篇更新日期为 2026-06-24。
- 事实要点：
  - Figma 同日发布 `Figma Motion`、设计 Agent 上下文增强、`Code on the Figma canvas`、`Connecting Figma and Weave` 等更新。
  - 官方表述强调 Motion 直接在同一文件里工作，设计 Agent 支持 custom tools、greater context 和 skills。
  - 这说明 Figma 的方向不是做更多“单次生成”，而是把创意、代码、动效、AI 协作收回到同一画布。
- 对创作者的意义：
  - 对 UI/UX、前端博主、设计教育、产品经理内容，这是明显的大选题。
  - 适合讨论“设计师以后是不是要在画布里直接调 Agent 和代码”。
- 可讲述角度：
  - 设计工具为什么开始吞并前端和动效工作流。
  - 设计 Agent 为什么必须有上下文和团队技能，而不是只会出草图。
- 风险与不确定性：
  - 今天能确认的是官方产品方向和功能发布；具体实际效率提升幅度还要等真实团队案例。
  - 这是一组更新，不宜把某一条拆成夸张结论。

### 3. Claude Tag：AI 正在从“个人助手”变成“频道成员”

- 时效性：高。Anthropic 官方新闻页日期为 2026-06-23。
- 事实要点：
  - Claude Tag 首发在 Slack，可被加入授权频道，任何成员都能 `@Claude` 委派任务。
  - Anthropic 明确写到 Claude 会基于频道内容积累相关上下文，并能规划未来任务。
  - 当前为 Claude Enterprise 和 Team 客户 beta。
- 对创作者的意义：
  - 这条最容易让普通读者一眼理解，因为“AI 进 Slack 当同事”本身就很有画面感。
  - 对做社群运营、知识付费团队、AI 管理升级的内容特别友好。
- 可讲述角度：
  - 为什么 AI 的下一场竞争是“谁占住团队工作流入口”。
  - 频道共享上下文会怎样改变交接、协作和复盘。
- 风险与不确定性：
  - 当前仍是 Slack 首发和 beta 阶段。
  - Anthropic 自述的内部使用效果可以引用，但不应外推成行业平均值。

### 4. GitHub Copilot CLI GA：终端开始长成完整的 Agent 工作台

- 时效性：高。GitHub changelog 日期为 2026-06-23。
- 事实要点：
  - GitHub Copilot CLI 新终端界面正式 GA。
  - 官方写明终端内可以浏览 Issues、Pull requests、gists。
  - 会话内可直接配置 MCP servers、skills、plugins、settings，而不是手改配置文件。
- 对创作者的意义：
  - 这是极适合做录屏、实操教程、生产力对比内容的热点。
  - 对开发者观众，它比“AI 帮我写个函数”更接近下一代工作方式。
- 可讲述角度：
  - 为什么 Terminal 会变成 AI 时代的新浏览器。
  - 终端、仓库、MCP、插件被统一之后，教程产品怎么做。
- 风险与不确定性：
  - 受众偏开发者，如果做泛流量内容，需要把术语翻译成“工作台升级”。

### 5. GitHub BYOK：多模型时代从口号变成默认配置

- 时效性：高。GitHub changelog 日期为 2026-06-23。
- 事实要点：
  - Copilot app 支持 BYOK，可接 OpenAI、Azure OpenAI、Anthropic、LM Studio、Ollama 和 OpenAI-compatible endpoint。
  - 官方强调密钥保存在本地 OS keychain，UI 不会读回。
  - 官方卖点明确指向 billing、quotas、regions、data-handling terms 与 tenant 内流量保留。
- 对创作者的意义：
  - 这条适合讲“以后不是选一家模型，而是组织如何调度多种模型”。
  - 对企业顾问、自媒体培训作者，这比单纯模型横评更贴近预算和决策。
- 可讲述角度：
  - BYOK 为什么是企业真正关心的 Agent 基建。
  - 私有网关、本地模型、前沿模型会如何混搭。
- 风险与不确定性：
  - 这是 Copilot app 范围内的能力，不等于所有开发环境今天都完全统一。

### 6. Runway + Figma Weave：视频改稿正在变得像设计改稿

- 时效性：高。Runway 官方页面日期为 2026-06-22。
- 事实要点：
  - Aleph 2.0 已接入 Figma Weave。
  - 官方写明可通过 keyframes 抽帧、改帧，再把修改同步回视频。
  - 支持最长 30 秒、1080p、多镜头序列编辑。
- 对创作者的意义：
  - 对短视频团队、品牌内容工作室、营销视频团队，这条非常接近日常生产场景。
  - 核心价值不在“从零生成更酷”，而在“少返工、少重拍、少切工具”。
- 可讲述角度：
  - 关键帧编辑会不会比整段重生成更快进入主流。
  - 设计画布为什么会吞掉视频后期的一部分流程。
- 风险与不确定性：
  - 复杂镜头和长时长场景下的稳定性，还需要更多外部样例验证。

### 7. ElevenLabs Ads Engine：AI 创作不再停在素材层，而是直接走向投放层

- 时效性：高。ElevenLabs 官方博客日期为 2026-06-22。
- 事实要点：
  - Ads Engine 是 ElevenCreative 内的新产品区。
  - 首个能力是广告本地化：读取现有广告素材与表现数据，执行文本翻译、图像适配、视频 dubbing，再推回 Google、Meta、LinkedIn。
  - 官方叙事重点是把“已经有效的广告”扩展到更多语言和市场。
- 对创作者的意义：
  - 对跨境电商、品牌代运营、内容出海博主，这是很强的商业化选题。
  - 它比通用配音工具更贴近 ROI，因此更容易引发付费用户和从业者关注。
- 可讲述角度：
  - AI 为什么开始从“做素材”走向“接渠道”。
  - 出海广告本地化为什么会成为新一轮内容工具战场。
- 风险与不确定性：
  - 是否提升投放效果仍取决于素材、预算、市场与账号策略，不能把平台能力直接写成结果承诺。

## 推荐选题

### 选题 01｜优先级 S

- 标题方向：`AI 同事真的来了：Claude 进 Slack 之后，团队协作为什么会先被重写`
- 目标受众：AI 工具关注者、知识付费操盘手、内容团队负责人、企业培训与咨询从业者
- 切题角度：
  - 不讲“Claude 又发布了什么模型”，只讲“AI 为什么开始占据团队沟通入口”。
  - 把 Claude Tag 放在 Slack、飞书、企业微信这类工作面争夺战里看。
- 爆点：
  - AI 不在私聊里，而在频道里。
  - 共享上下文、异步执行、可 `@`、可治理，这些词比“更强模型”更容易传播。
- 内容结构：
  1. 先讲一句话新闻：Anthropic 把 Claude 变成 Slack 里的团队成员。
  2. 解释为什么这代表 AI 从个人助手升级为组织协作者。
  3. 结合创作者场景：内容选题群、剪辑协作群、课程交付群、客服群。
  4. 讲边界：权限、预算、误触发、错误上下文。
  5. 收束到判断：下一轮工具战是抢团队入口，不是只比模型分数。
- 痛点：
  - 团队信息分散，重复解释上下文。
  - 内容生产交接成本高，任务总在群里烂尾。
- 爽点：
  - AI 能直接在真实工作流里接任务，而不是每个人各自开一个聊天窗。
- 痒点：
  - 大家都在问“AI 员工什么时候来”，这个就是最接近的产品形态。
- 推荐内容形式：公众号深度稿、口播评论、直播拆解、社群分享 PPT
- 可引用热点来源：
  - Anthropic：<https://www.anthropic.com/news/introducing-claude-tag>
  - GitHub BYOK：<https://github.blog/changelog/2026-06-23-github-copilot-app-support-for-byok/>

### 选题 03｜优先级 A

- 标题方向：`浏览器 Agent 进入公测：Gemini Computer Use 离“自动做事”还有几步`
- 目标受众：Agent 创业者、自动化工具开发者、教程型自媒体、效率博主
- 切题角度：
  - 不做泛泛新闻搬运，重点讲“执行层”。
  - 用“会不会成为下一波自动化产品底座”来切。

  - 还把安全政策和 prompt injection detection 写进发布说明里。
- 内容结构：
  1. 解释 `Computer Use` 到底是什么，不要让非技术观众误解。
  2. 对比“会回答”和“会操作”的差别。
  3. 讲适合落地的场景：信息搜集、内容搬运、后台录入、运营 SOP。
  4. 讲暂时别冲的场景：高风险支付、复杂多跳登录、强合规动作。
  5. 收束到“Agent 产品化的关键不是演示，而是稳定和边界”。

  - 现在很多自动化流程仍需人工点网页、切应用、确认状态。

- 痒点：
  - 观众最爱问“AI 什么时候能直接替我点完这些步骤”。
- 推荐内容形式：实操解读视频、教程文章、直播演示、训练营选题
- 可引用热点来源：
  - Gemini API changelog：<https://ai.google.dev/gemini-api/docs/changelog>

## 今日最推荐选题

`AI 同事真的来了：Claude 进 Slack 之后，团队协作为什么会先被重写`

原因很直接：它既有强新闻感，也有强代入感，还能自然带出 Agent、组织记忆、工作入口争夺和知识付费团队升级这几条更大的叙事线。比单纯聊模型参数，更容易做出传播和转化。
