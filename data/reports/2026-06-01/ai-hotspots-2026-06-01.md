## 选题推荐（3–5 个，含优先级）

### 选题 01（P0，最推荐）：OpenAI 模型/能力变动 → 创作者 30–90 天迁移清单（附回归题库）

- 标题方向（可二选一）
  - 《OpenAI 又改模型了：创作者 30 天“保命迁移清单”》
  - 《同一提示词输出变了？用一套回归题库把风险降到最低》
- 目标受众：AI 自媒体创作者、短视频脚本作者、做知识付费/训练营的人、用 AI 做商业交付的个人/小团队
- 切题角度：把“平台更新”翻译成“你今天必须做的 7 件事”
- 爆点：输出“可直接抄”的回归题库（10–20 个问题）+ 迁移对照表（旧模型/新模型怎么选）
- 痛点：同一提示词不稳定、交付风格漂移、团队协作难对齐
- 爽点：一张清单把“不可控”变“可控”，把焦虑变流程
- 痒点：观众想知道“我到底该用哪个模型/怎么设置才稳”
- 推荐形式：横屏信息密度视频 / 长图文 / 直播带表格讲解
- 内容结构（建议 6 段）
  1. 发生了什么：引用官方 release notes（只讲事实）
  2. 影响谁：脚本/配音/海报/课程交付分别怎么受影响
  3. 三类风险：风格漂移 / 事实性错误 / 成本与速度波动
  4. 迁移清单：工具设置、提示词策略、采样/温度、结构化输出
  5. 回归题库：固定 10–20 个用例（附评分标准）
  6. 结论：你该怎么选 + 一周内怎么复盘
- 可引用来源：
  - https://help.openai.com/en/articles/6825453-chatgpt-release-notes
- 风险与不确定性：
  - 具体退役时间/适用产品线以官方页面为准；不要把“推测的产品策略”说成事实。

### 选题 02（P1）：Claude Opus 4.8 到底强在哪？用“同题同资料源”的工作流对照测评

- 标题方向
  - 《Opus 4.8 发布：我用 3 个真实任务测到的提升/短板》
- 目标受众：做长文、研究解读、代码类内容的创作者；企业内训/知识付费讲师
- 切题角度：拒绝跑分，直接跑“你的工作流”
- 爆点：同一资料包（PDF/网页/代码仓）三模型输出对照，一眼看差距
- 推荐形式：图文测评 + 直播复盘（观众能带任务上麦）
- 内容结构（建议 5 段）
  1. 官方说了什么（摘关键点并给链接）
  2. 测评协议：同输入、同约束、同评分标准
  3. 任务 1：长文结构化（摘要/大纲/观点/引用）
  4. 任务 2：代码/数据分析（可复现）
  5. 结论：适合哪些创作者、什么场景别用
- 可引用来源：
  - https://www.anthropic.com/news/claude-opus-4-8
- 风险与不确定性：
  - 模型表现受提示词和上下文策略影响，需公开测评设置，避免误导。

### 选题 03（P1）：网页变更监控 + Agent 自动总结：把“信息差”做成可复用的创作机器

- 标题方向
  - 《我把 20 个 AI 信息源接入“监控→总结→选题”流水线，日更不再靠刷推》
- 目标受众：日更型自媒体、资讯号、做行业周报/训练营的人
- 切题角度：工具链拆解（信息源清单 + 触发条件 + 产出格式）
- 爆点：公开你的“监控清单”（官方 blog / GitHub releases / 论文 RSS）
- 推荐形式：教程视频 + GitHub 模板仓 + 直播答疑
- 内容结构（建议 6 段）
  1. 为什么要监控：刷信息流的时间成本与遗漏风险
  2. 信息源分类：官方/开源/媒体/社区
  3. 监控策略：频率、关键词、去重、阈值（什么变化才通知）
  4. Agent 输出：摘要模板、风险标注、可引用链接列表
  5. 选题生成：按受众拆解（开发者/创作者/老板）
  6. 复盘：误报/漏报如何调参
- 可引用来源：
  - https://firecrawl.dev/blog/monitor
- 风险与不确定性：
  - 自动化抓取需遵守目标站点条款；对采集失败要有 fallback（人工检查）。

### 选题 04（P2）：开源 Agent 框架怎么选（Hermes-Agent 为例）：从“能跑”到“可用”差在哪

- 标题方向
  - 《别再只看 star：我用 3 个指标筛开源 Agent 框架》
- 目标受众：做工具号、开发者向内容、做 Agent 课程/训练营的人
- 切题角度：对比“集成成本/可观测性/稳定性”，不卷概念
- 推荐形式：图文对照 + demo 录屏
- 可引用来源：
  - https://github.com/NousResearch/hermes-agent/releases
- 风险与不确定性：
  - release 频率与质量不等价；建议把“主观体验”与“可验证指标”分开讲。

### 选题 05（P2）：FiftyOne Agent：把素材库管理变成“可对话的数据工作台”

- 标题方向
  - 《素材太多找不到？我用 Agent 把 2TB 视频库变成可搜索的“镜头百科”》
- 目标受众：视频创作者、多模态应用开发者、做数据标注/检索的人
- 切题角度：用一个真实素材库任务演示（找相似镜头/自动打标签/筛选片段）
- 推荐形式：实操视频 + 数据集/脚本公开（可脱敏）
- 可引用来源：
  - https://voxel51.com/blog/introducing-fiftyone-agent/
- 风险与不确定性：
  - 需要准备数据与环境；演示时注意版权与隐私素材脱敏。
